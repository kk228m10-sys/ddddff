# No Skip Club — Telegram-бот

Бот для проекта **No Skip Club** («Дисциплина или заплати»): онбординг с вопросами,
главное меню, план тренировок на неделю по уровню, библиотека упражнений с техникой,
каталог видео (sportkuznica.com), подтверждение тренировки **тренером по фото**,
питание, трекинг стрика, напоминания в тренировочные дни, запись на созвон.
Мини-приложение с персонажем временно скрыто (`CHARACTER_ENABLED=false`).

Всё крутится **у тебя на компьютере**, без подписок и облака. Нужен только
Python и бесплатный токен бота от Telegram.

---

## 1. Что понадобится

- Python 3.10+ (проверить: `python3 --version`)
- Telegram-аккаунт
- ~10 минут на первый запуск

---

## 2. Установка

```bash
cd no_skip_club_bot
python3 -m venv venv
source venv/bin/activate        # на Windows: venv\Scripts\activate
pip install -r requirements.txt
```

---

## 3. Получаем токен бота

1. В Telegram напиши **@BotFather**
2. Команда `/newbot`, придумай имя и username (должен заканчиваться на `bot`)
3. Скопируй токен вида `123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

Узнать свой личный `chat_id` (нужен для заявок на созвон): напиши **@userinfobot**,
он пришлёт твой id.

---

## 4. Настройка

```bash
cp .env.example .env
```

Открой `.env` и впиши:

```
BOT_TOKEN=твой_токен_от_BotFather
ADMIN_CHAT_ID=твой_chat_id
```

`WEBAPP_URL` пока оставь как есть — заполним на шаге 6.

---

## 5. Запуск бота (без мини-аппа)

```bash
python3 bot.py
```

Если увидел `Бот запущен` — открывай своего бота в Telegram и жми `/start`.

**Клиент:** план на неделю → «🎬 Видео техники на сегодня» → после тренировки
«📸 Подтвердить тренировку» (фото уходит админу).

**Админ-команды:** `/pending` (заявки на зачёт), `/unlock <id>`, `/blocked`,
`/addmaterial` (ответом на фото/видео).

База данных `no_skip.db` создастся автоматически рядом с ботом — там хранится всё:
пользователи, планы, история тренировок, заявки на созвон.

---

## 6. Запуск мини-аппа с персонажем

Мини-апп — отдельный веб-сервер. Telegram может открыть его внутри бота **только
по HTTPS-адресу**, поэтому локальный сервер нужно "пробросить" в интернет через
туннель. Проще всего — **ngrok** (бесплатно).

### 6.1. Установи ngrok

Скачай с https://ngrok.com/download (есть версии под Windows/Mac/Linux),
зарегистрируйся бесплатно и подключи свой аккаунт (команда будет на их сайте, тип
`ngrok config add-authtoken <твой_токен>`).

### 6.2. Запусти веб-сервер мини-аппа

В отдельном терминале (бот из шага 5 пусть продолжает работать):

```bash
source venv/bin/activate
python3 webapp/server.py
```

Сервер поднимется на `http://127.0.0.1:5000`.

### 6.3. Пробрось его в интернет

Ещё в одном терминале:

```bash
ngrok http 5000
```

Ngrok покажет строку вида:

```
Forwarding   https://a1b2c3d4.ngrok-free.app -> http://localhost:5000
```

Скопируй этот `https://...ngrok-free.app` адрес.

### 6.4. Пропиши адрес в .env

Открой `.env`, вставь адрес:

```
WEBAPP_URL=https://a1b2c3d4.ngrok-free.app
```

Сохрани и **перезапусти бота** (Ctrl+C в терминале с `bot.py`, затем снова
`python3 bot.py`). Теперь в главном меню появится кнопка «🎮 Мой персонаж».

> ⚠️ У бесплатного ngrok адрес меняется при каждом перезапуске туннеля. Значит,
> при каждом новом запуске нужно будет обновлять `WEBAPP_URL` в `.env` и
> перезапускать бота. Если это надоест — можно взять платный статический домен
> в ngrok или задеплоить `webapp/server.py` на любой бесплатный хостинг
> (Render, Railway, PythonAnywhere) — тогда адрес будет постоянным.

---

## 7. Как всё это использовать каждый день

Чтобы бот и персонаж работали одновременно, держи открытыми **три терминала**:

1. `python3 bot.py` — сам бот
2. `python3 webapp/server.py` — сервер мини-аппа
3. `ngrok http 5000` — туннель в интернет

Если тебе не нужен персонаж прямо сейчас — достаточно только терминала 1,
бот будет работать полностью, кроме кнопки «Мой персонаж».

## 9. Как редактировать контент

Всё живёт в `content.py` — обычные словари и списки, код трогать не нужно:

- `EXERCISE_LIBRARY` — упражнения и техника по уровням
- `NUTRITION_TIPS` — советы по питанию по целям
- `ABOUT_TEXT` — текст раздела "О проекте"
- `CHARACTER_STAGES` (в `database.py`) — пороги стрика и названия стадий персонажа

## 10. Онбординг

Порядок вопросов при первом запуске:

1. Уровень подготовки (новичок / средний / продвинутый).
2. Дни тренировок — от 3 до 7 дней в неделю, конкретные дни выбираются вручную.
3. Место — дома или в зале.
4. Вид тренировок — Похудение/рельеф, Сила/масса, Выносливость/координация,
   Взрывная сила, ОФП. Можно выбрать и совместить не более двух видов.
5. Обещание себе.

Вид тренировок можно поменять в любой момент через кнопку «🔄 Сменить вид
тренировок» в главном меню — план на текущую неделю пересоберётся под новые
виды. Место и уровень пока меняются только через прямое обновление в базе
(или переустановку бота).

## 11. Логика стрика, блокировки и оплаты

- Каждую неделю (с понедельника) генерируется новый план тренировок под
  уровень, место (дома/зал) и виды тренировок пользователя.
- В тренировочный день бот в `REMINDER_HOUR` (по умолчанию 8:00) присылает
  напоминание всем, кто ещё не отметил тренировку сегодня (кроме
  заблокированных).
- В 23:50 бот проверяет: если сегодня был тренировочный день, а тренировка не
  отмечена — стрик обнуляется, пользователь **блокируется** и теряет доступ
  к боту.
  - Пользователю приходит сообщение с суммой штрафа и ссылкой на оплату
    (`PENALTY_AMOUNT_RUB`/`PENALTY_AMOUNT_KZT` и `COACH_USERNAME` в `.env`).
  - Тебе как админу (`ADMIN_CHAT_ID`) приходит уведомление: кто именно и
    сколько должен, с готовой командой для разблокировки.
- Пока пользователь заблокирован, любые его сообщения и нажатия кнопок в
  боте перехватываются, и он получает напоминание про оплату (см.
  `handlers/blocking.py`).
- После получения оплаты **вручную** разблокируй командой:
  ```
  /unlock <telegram_id>
  ```
  Список всех заблокированных сейчас — командой `/blocked`. Обе команды
  работают только у тебя (`ADMIN_CHAT_ID`).
- Стадии персонажа в мини-аппе: Спящий (0) → Пробуждение (1) → Новобранец (3) →
  Боец (7) → Дисциплина (14) → Несгибаемый (28 дней). Пороги легко менять в
  `CHARACTER_STAGES` в `database.py`.

## 11. Частые проблемы

**Бот не отвечает на /start**
Проверь, что в `.env` правильный `BOT_TOKEN` и что терминал с `python3 bot.py`
не выдал ошибку при запуске.

**Кнопка "Мой персонаж" не появляется**
Она показывается только если в `.env` заполнен `WEBAPP_URL`. После заполнения
обязательно перезапусти `bot.py`.

**Мини-апп открывается, но пишет "Не удалось загрузить данные"**
Убедись, что `webapp/server.py` запущен и что ngrok-туннель активен и указывает
именно на порт из `WEBAPP_PORT` (по умолчанию 5000).

**Заявки на созвон не приходят**
Проверь `ADMIN_CHAT_ID` в `.env` — это должен быть твой личный numeric id
(узнать через @userinfobot), а не username.

## 12. Новое: вес, материалы, запись по слотам, ИИ-психолог

- **Вес** — спрашивается на онбординге сразу после уровня подготовки, хранится в
  профиле и виден в «Мой прогресс».
- **Упражнения и техника** — теперь отдельная кнопка, показывает текстовую
  библиотеку сразу под текущий уровень и место пользователя, без повторного
  выбора.
- **Материалы (фото/видео)** — отдельная кнопка. Изначально пусто, наполняешь
  сам командой в чате с ботом (от твоего аккаунта, `ADMIN_CHAT_ID`), **ответив**
  на сообщение с фото или видео:
  ```
  /addmaterial <место|all> <уровень|all> <подпись>
  ```
  Например: `/addmaterial home beginner Техника приседаний`. `all` — показывать
  всем независимо от места/уровня. Пользователь в разделе «Материалы» видит
  только то, что подходит под его текущие место и уровень (плюс всё с `all`).
- **Запись на созвон** — теперь не свободный текст, а фиксированные слоты:
  среда и суббота, 17:00–20:00, по часу, максимум 10 человек на слот. При
  записи тебе приходит уведомление с числом занятых мест. У пользователя есть
  «Мои записи» с возможностью отменить.
- **ИИ-психолог** — кнопка в меню, использует бесплатный Mistral AI. Нужно
  получить ключ на console.mistral.ai и вписать в `.env`:
  ```
  MISTRAL_API_KEY=...
  MISTRAL_MODEL=mistral-small-latest
  ```
  Промпт настроен так, чтобы бот поддерживал человека, искал мягкий компромисс
  по тренировкам, но всегда подводил к тому, что тренировки продолжать нужно, а
  штраф за пропуск — это справедливо. Без ключа кнопка вежливо сообщает, что
  функция не настроена, остальной бот работает как обычно.
- **Меню всегда под рукой** — внизу экрана есть постоянная кнопка «☰ Меню», а
  также команда `/menu` рядом с полем ввода (через список команд бота).

## 13. Фото-подтверждение тренировки («жест дня»)

Каждое утро в тренировочный день бот вместе с напоминанием выбирает случайный
жест руки (✌️👍🤙✊🤟👌🖐) — один и тот же для всех на весь день. Пользователь
должен сфотографироваться в зале/дома, показывая этот жест и обстановку
вокруг, и прислать фото через кнопку «📸 Подтвердить тренировку (фото)» — она
есть и в главном меню, и в карточке дня.

Фото проверяет отдельный ИИ (тоже Mistral, но с моделью, понимающей
изображения — `MISTRAL_VISION_MODEL`, по умолчанию `pixtral-12b-2409`). Ему
задан промпт **строго проверяющего тренера**: он смотрит, виден ли на фото
нужный жест и похожа ли обстановка на настоящий зал/дом (а не на
ИИ-сгенерированную картинку, скриншот или чужое фото), и отвечает только
«ПОЛОЖИТЕЛЬНО» или «ОТРИЦАТЕЛЬНО» — без единого лишнего слова. Он не ведёт
диалог и не может быть переубеждён пользователем в переписке, в отличие от
ИИ-психолога — это два разных промпта и два независимых вызова API.

- При положительном вердикте тренировка автоматически засчитывается, стрик
  растёт как обычно.
- При отрицательном — бот просит переснять, тренировка не засчитывается, но
  штраф/блокировка не срабатывают немедленно: если до конца дня так и не
  придёт удачное фото (или отметка «✅ Отметить без фото»), вечерняя проверка
  сработает как раньше и заблокирует пользователя за пропуск.
- Ручная отметка «✅ Отметить без фото» по-прежнему доступна — фото не
  обязательно, это просто более надёжный способ подтверждения на случай, если
  вдруг возникнут споры о честности стрика.
- Используется тот же `MISTRAL_API_KEY`, что и для ИИ-психолога. Без ключа
  кнопка вежливо сообщает, что проверка недоступна.
